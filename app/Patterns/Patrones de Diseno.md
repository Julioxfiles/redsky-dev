
# Patrones de Diseño

Los patrones de diseño son soluciones reutilizables y probadas a problemas comunes en el diseño de software. No son código listo para usar, sino guías o estructuras que ayudan a escribir sistemas más organizados, flexibles y mantenibles.

---
# 1. Patrones Creacionales

Son aquellos que se encargan de **cómo se crean los objetos** dentro de un sistema.

Su objetivo principal es:

* Evitar la creación directa y rígida de objetos
* Hacer el código más flexible al momento de instanciar
* Permitir cambiar *qué se crea* sin afectar el resto del sistema

En otras palabras:

Controlan el proceso de creación para que el sistema no dependa de clases concretas.

---

# 2. Patrones Estructurales

Se enfocan en **cómo se organizan y relacionan las clases y objetos**.
Su objetivo principal es:

* Construir estructuras más grandes a partir de piezas pequeñas
* Facilitar la reutilización
* Reducir el acoplamiento entre componentes

En otras palabras:

Definen cómo encajan las piezas del sistema para formar algo más complejo.

---

# 🔄 3. Patrones de Comportamiento

Se centran en **cómo los objetos interactúan y se comunican entre sí**.

Su objetivo principal es:

* Definir responsabilidades entre objetos
* Gestionar el flujo de información
* Hacer la comunicación más flexible y desacoplada

En otras palabras:

Regulan cómo colaboran los objetos para ejecutar la lógica del sistema.

---

# Resumen simple

* **Creacionales** → creación de objetos
* **Estructurales** → organización de objetos
* **Comportamiento** → comunicación entre objetos
